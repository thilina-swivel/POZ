The Problem

A single color choice
can break the interface

When a client sets white as their brand color, buttons become invisible. The current system has no way to adapt — it simply applies the color and hopes for the best.

How It Works

Three steps.
Fully automatic.

When a client saves their brand color, the system handles everything behind the scenes. No manual work from the team.

1
Client picks their brand color
From the Company Settings page, the client chooses a primary and secondary color using a color picker. The system checks the color before it's saved.
BrandingForm — Color Picker
2
The system calculates the right text color
Using a standard accessibility formula, the system checks how light or dark the brand color is — then automatically decides whether text on top should be black or white to stay readable.
Accessibility Formula (WCAG 2.1)  ↗ Read the standard
3
Every button, badge, and label updates instantly
The calculated text color is applied across the entire interface automatically. Buttons, navigation tabs, status badges — all adapt to the client's brand color without anyone touching a line of code.
Applied across the whole portal

Color Safety Rules

Three clear outcomes
for every color choice

The system evaluates any color the client picks and gives an immediate result. There are no surprises — the interface either approves, warns, or blocks the choice before anything is saved.

✓
Approved
The color has a strong contrast score (4.5 or higher). Text and icons on top will be clearly readable for all users, including those with visual impairments.
→ Client can save freely
⚠
Warning
The color is marginal (score 3.0 to 4.5). Most users can read it fine, but it may be harder for users with low vision. A warning message is shown before saving.
→ Client can save with a warning
✕
Blocked
The color would make text unreadable (score below 3.0). White is a common example. The Save button is disabled and an explanation is shown inline.
→ Client must choose a different color
What Changes

Small changes.
Big impact.

Only four parts of the product need to be updated. Everything else — the backend, database, charts, and most UI elements — stays exactly the same.

Needs update Brand color used as background with text on top — must apply auto text color

Sidebar — Active Nav Items Main navigation, left sidebar Active item uses brand color background with fixed white text — breaks on light colors Apply auto text color
Primary Buttons Throughout — Save, Submit, View, Export PDF Hardcoded text-white — invisible when brand color is white or near-white Apply auto text color
